# Rules — Always-on project rules

These rules are loaded automatically during every session. They guide the AI to work safely and consistently.

## What's here
| File | What it does |
|---|---|
| `critical-actions.md` | Stops dangerous commands (like `rm -rf` or `git push --force`) so you can review them first. |
| `security.md` | Ensures safe coding practices (e.g., no hardcoded secrets). |
| `code-style.md` | Keeps the code style consistent across the project. |

You don't need to run any commands to use these; they are always active.
